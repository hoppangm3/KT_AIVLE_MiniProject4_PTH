# 📚 [요약정리] 에이블스쿨 4차 미니프로젝트: AI 도서관리시스템 개발

## 1. 프로젝트 개요
*   **주제:** AI(OpenAI)를 활용하여 도서 표지 이미지를 자동으로 생성하는 기능을 포함한 **프론트엔드 중심 도서 관리 시스템** 개발
*   **서비스명:** "걷기가 서재" (작가의 산책 서비스)
*   **목표:** React + fetch API를 통한 CRUD 구현 실습 및 외부 AI API 연동 경험 확보

## 2. 핵심 기술 스택
*   **Frontend:** React 19 (Vite 기반), fetch API (데이터 통신)
*   **Backend:** `json-server` (로컬 REST API 역할을 하는 가짜 서버)
*   **AI:** OpenAI API (GPT Image 모델 - 이미지 생성)
*   **UI Library:** MUI (Material UI) 권장 (생산성 및 디자인 일관성 확보)

## 3. 주요 기능 (요구사항)
1.  **도서 목록 확인:** 등록된 도서의 제목, 등록일 등을 리스트나 카드 형태로 확인
2.  **신규 도서 등록:** 제목과 내용을 입력하여 새로운 도서 추가 (유효성 검사 포함)
3.  **상세 정보 조회:** 도서 클릭 시 본문 내용 및 표지 이미지 확인
4.  **수정 및 삭제:** 기존 도서 정보 수정 및 필요 없는 도서 삭제
5.  **AI 표지 생성 (핵심):** 도서 내용을 기반으로 AI가 어울리는 이미지를 자동 생성하고 저장

## 4. 3일간의 진행 일정 (Step-by-Step)
| 일차 | 주요 목표 | 상세 활동 |
| :--- | :--- | :--- |
| **1일차** | **기획 및 UI 뼈대** | R&R 분담, `db.json` 설계, Vite+React 프로젝트 생성, Mock 데이터 UI 구현 |
| **2일차** | **CRUD 기능 완성** | fetch(GET/POST/PATCH/DELETE) 연동, 폼 상태 관리 및 불변성 패턴 적용 |
| **3일차** | **AI 연동 및 발표** | OpenAI API 연동(표지 생성), PATCH 요청으로 이미지 저장, 최종 발표 및 리뷰 |

## 5. 기술적 핵심 포인트 (Hint)

### ① 데이터 구조 (db.json)
```json
{
  "books": [
    {
      "id": 1,
      "title": "도서 제목",
      "author": "저자",
      "content": "도서 내용 (AI 프롬프트 생성용)",
      "coverImageUrl": "", // AI 생성 후 Data URL 저장
      "createdAt": "ISO 날짜",
      "updatedAt": "ISO 날짜"
    }
  ]
}
```

### ② OpenAI API 연동 흐름
1.  **요청:** React에서 도서 제목/내용을 조합한 `prompt`와 `API Key`를 담아 OpenAI에 `POST` 요청
2.  **응답:** OpenAI가 `b64_json`(Base64 문자열) 형태로 이미지 데이터 반환
3.  **변환:** React에서 이를 `Data URL`(`data:image/png;base64,...`) 형태로 가공
4.  **저장:** `json-server`에 `PATCH` 요청을 보내 해당 도서의 `coverImageUrl` 필드만 업데이트

### ③ MUI 활용 (부록 내용)
*   **Box:** 스타일 약어(p, mt 등)를 사용하여 CSS 작성 없이 컴포넌트 내부에서 디자인 제어
*   **Button:** `variant="contained"` 등 속성 하나로 완성된 스타일 적용
*   **AppBar:** 복잡한 CSS 없이 상단 네비게이션 바를 손쉽게 구현
